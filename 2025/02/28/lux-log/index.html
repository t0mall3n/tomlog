Introduction

Neural networks are very promising for studying actual synaptic behavior in specific human tasks, as they directly mimic neural communications in our brains.

Julia is an excellent choice for implementing neural network simulations due to its flexibility (similar to Python, but with flexible and strict type annotations), JIT compilation for fast computing, and abundant libraries for machine learning.

This article aims to introduce Lux.jl and other libraries primarily as a reference for myself in the future, focusing on the essential steps for creating neural network simulations.

Environment

  • Ubuntu(WSL) on Windows11
  • Julia v1.11.3

Initialize Parameters and States

Unlike Flux.jl, Lux.jl models only define the computation flow - parameters and states are not included in the model definitions. This separation provides more flexibility and control.

You need to generate the parameters and states using Lux.setup before running models. When executing your model, you must provide the input, parameters and states.

The model forward function returns both output and new_state, and the state needs to be updated at a certain appropriate times during training or inference. With the given seed, you might get an output like Float32[5.2590766, 3.3526256, 5.555236, -1.923615].

Optimizer

Now we have obtained gradients of the parameters for the MSE loss. To apply these gradients, we need to prepare an optimizer using Optimisers.jl. Let’s use the Adam optimizer. If you’re familiar with optimization techniques, you might want to clip gradients for more stable learning. You can use ClipGrad or ClipNorm for this purpose, and chain them with the Adam optimizer using Optimisers.OptimiserChain.

Optimizers also need to be initialized with Optimisers.setup(opt, ps), which returns an “optimizer state” - a set of optimizers for your model parameters. Anyway, just remember you will use opt_state for the optimization process. To apply the optimization, run either Optimisers.update!(opt_state, ps, grads) for in-place updates or opt_state, ps = Optimisers.update(opt_state, ps, grads) for the explicit notation.
